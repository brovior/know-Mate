# UI_SPEC.md — Aegis Desk 화면 사양

> 룩앤필 시각 레퍼런스는 app/ui/mockup.html 을 기준으로 삼는다.
> 이 문서는 수치·색상·동작 규칙을 정의하고, mockup.html은 실제 모양을 보여준다.
> 둘이 충돌하면 이 문서의 수치·동작 규칙을 우선한다.

## 1. 설계 원칙

**UI는 셸(shell), 에이전트는 플러그인이다.**
채팅 영역·입력바·렌더러는 에이전트 종류와 무관한 공통 컴포넌트이고,
에이전트(지식검색, 향후 MES 에이전트 등)는 표준 응답 블록(JSON)만 반환한다.
새 에이전트 추가 시 UI 코드 수정 없이 사이드바 패널 컴포넌트와
백엔드 등록만으로 동작해야 한다. (CLAUDE.md 4장 멀티 에이전트 원칙과 일치)

## 2. 레이아웃

```
┌────────────────────────────────────────────────┐
│ 타이틀바 (h:52) [로고] Aegis Desk | 우측: 사내망 전용·사용자명 │
├──────────┬─────────────────────────────────────┤
│ 사이드바   │ 탭바 (스레드 탭 + 새 대화)              │
│ (w:280)  ├─────────────────────────────────────┤
│  모드전환  │ 채팅 영역 (스크롤)                      │
│  모드패널  │   사용자 말풍선(우측) / AI 응답(좌측)      │
│  최근질문  │   AI 응답 = 블록 렌더러 출력             │
│  인덱싱    ├─────────────────────────────────────┤
│  새 대화   │ 입력바 (h:56, 알약형) + [전송] 버튼      │
└──────────┴─────────────────────────────────────┘
```

사이드바 구성 (위→아래):
1. **모드 전환 세그먼트** (상단 고정) — 지식검색 / MES 에이전트
2. **모드별 패널** (에이전트가 제공)
   - 지식검색: 검색 범위 체크박스 (내 PC 문서 / 공유 폴더 / 내 메일함 + 건수)
   - MES 에이전트: 빈 패널 + "준비 중" 표시
3. **최근 질문 목록** (공통, 스크롤 영역)
4. **인덱싱 상태 카드 + 버튼 2개** (하단 고정) — 7장 참조
5. **새 대화 버튼** (최하단 고정)

- 라운드: 말풍선 16px, 카드 10px, 입력바·버튼 28px(알약)

## 4. 에이전트 응답 블록 스펙 (핵심)

에이전트 백엔드는 아래 JSON을 반환하고, 프런트 렌더러는 type별로 그린다.
Block 타입 정의는 CLAUDE.md 4-1 (agents/base.py TypedDict)과 1:1 대응한다.

```json
{
  "blocks": [
    { "type": "text",
      "content": "2025년 3월 A설비 알람 폭주 건은..." },

    { "type": "sources",
      "title": "관련 문서 3건 · 근거 자료",
      "items": [
        { "badge": "메일",
          "title": "RE: A설비 알람 폭주 조치 결과 보고",
          "subtitle": "보낸사람: 김OO 프로 · 2025.03.14",
          "score": 0.95,
          "path": "C:/.../원본.msg" } ] },

    { "type": "table",
      "title": "설비별 알람 현황",
      "columns": ["설비", "알람수", "상태"],
      "rows": [["A설비", 42, "주의"]] },

    { "type": "chart",
      "chart_type": "line|bar",
      "title": "주간 알람 추이",
      "x": ["월","화","수"], "series": [{ "name": "A설비", "data": [3,5,2] }] }
  ]
}
```

렌더링 규칙:
- `text`: AI 말풍선 내 본문. 줄바꿈 유지
- `sources`: 카드 리스트. badge 색은 메일=primary-light, 문서=teal-badge.
  score는 "일치율 95%"로 우측 정렬. 카드 클릭 → path의 원본 파일 열기
  (`os.startfile`). 파일이 없으면 토스트로 "원본을 찾을 수 없음" 안내
- `table`/`chart`: MES 에이전트용. LLM이 이 스키마로 응답하면 UI가 동적 렌더링
- 알 수 없는 type은 무시하지 말고 "지원하지 않는 응답 형식" 회색 박스로 표시

## 5. 상호작용

- 전송: Enter(전송) / Shift+Enter(줄바꿈), 응답 대기 중 입력바 비활성 +
  로딩 인디케이터(점 3개 애니메이션)
- 모드 전환 시: 대화 스레드는 모드별로 분리 유지, 사이드바 패널 교체
- 최근 질문 클릭: 해당 스레드 복원 (threads.json에서 messages 로드 → 블록 렌더)
- 탭바: 열린 스레드 탭 + "새 대화 +" 탭

## 6. 검색 범위 패널 (지식검색 모드)

```
검색 범위
☑ 내 PC 문서      [2,841건]   ← teal 배지
☑ 공유 폴더        [359건]    ← teal 배지
☐ 내 메일함        [준비 중]   ← 회색 배지 (Phase 5까지 비활성)
```

- 체크박스 → retriever scope 필터와 연동 (CLAUDE.md 6-9)
  - 내 PC 문서 = scope:local / 공유 폴더 = scope:shared
- 메일함은 Phase 5까지 "준비 중" 회색 배지로 비활성 표시

## 7. 인덱싱 UI 명세

### 온보딩 화면 (최초 1회 — state.json 없을 때 자동 표시)
```
┌─────────────────────────────────────┐
│  Aegis Desk 첫 설정                  │
│                                     │
│  인덱싱할 폴더를 선택하세요            │
│  [C:/Users/문서        ] [✕]        │
│  [Z:/공유폴더          ] [✕]        │
│  [+ 폴더 추가]                       │
│                                     │
│  예상 문서 수: 약 3,200건            │
│                                     │
│  ████████████░░░░░░░  128/3,200건   │
│  A설비_점검매뉴얼.docx 처리 중...     │
│                                     │
│         [인덱싱 시작]  [취소]        │
└─────────────────────────────────────┘
```
- 진입: 앱 최초 실행 시 state.json 없으면 자동 표시
- 재진입: 사이드바 [폴더 관리] 버튼 → 동일 오버레이 (타이틀만 "폴더 관리")
- 폴더 추가/제거는 config.yaml과 양방향 동기화 (CLAUDE.md 6-11)

### 메인 화면 사이드바 하단 (인덱싱 상태 카드)
```
─────────────────────────────
✅ 마지막 인덱싱: 오늘 11:24
   문서 3,200건 / 폴더 2개
   [지금 재인덱싱]  [폴더 관리]
─────────────────────────────

↓ 재인덱싱 진행 중

─────────────────────────────
⟳ 인덱싱 중...
██████░░░░░░░░  45/128건
A설비_점검매뉴얼.docx 처리 중...
[취소]
─────────────────────────────
```

사이드바 하단 버튼 2개:
- **[지금 재인덱싱]**: 변경된 파일만 증분 처리. 클릭 시 사이드바 내
  프로그레스 바 표시 + 버튼이 [취소]로 전환
- **[폴더 관리]**: 온보딩과 동일 오버레이 오픈. 폴더 추가/제거 + 전체 재인덱싱

프로그레스 바 규칙:
- 표시 정보: `n/총건수` + **현재 처리 중인 파일명**
- 진행률은 수집기(QThread)에서 pyqtSignal로 전달받아 갱신
- 취소: 진행 중 언제든 중단 가능, 중단 시점까지 인덱싱된 내용은 유지
- 완료 후: "✅ 마지막 인덱싱: 방금 전" 상태로 복귀

## 8. 컴포넌트별 상태 정의 (구현 참조)

| 컴포넌트 | 상태 | 표시 |
|---|---|---|
| 입력바 | idle | 활성, 플레이스홀더 |
| 입력바 | 응답대기 | 비활성 + 로딩 인디케이터 |
| 인덱싱 카드 | idle | "✅ 마지막 인덱싱" + 버튼 2개 |
| 인덱싱 카드 | 진행중 | "⟳ 인덱싱 중" + 프로그레스 바 + [취소] |
| 메일 체크박스 | Phase 1~4 | "준비 중" 회색, 비활성 |
| MES 패널 | 전체 | "준비 중" placeholder |
| 소스 카드 | 클릭 | 파일 열기 / 없으면 토스트 |
| 응답 블록 | 알 수 없는 type | 회색 "지원하지 않는 형식" 박스 |
