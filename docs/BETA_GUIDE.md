# Aegis Desk 베타 테스터 가이드

> 버전: 0.9.0-beta2

## 0. beta2 변경 사항

- **DRM 보호 문서 인덱싱 지원**: beta1에서 일부 `.xlsx`/`.pptx`가 `File is not a zip
  file` / `Package not found`로 인덱싱 실패하던 문제를 해결했습니다. 사내 DRM으로
  보호된 문서는 이제 Office(Excel/PowerPoint)를 통해 자동으로 열어 인덱싱합니다.
  단, DRM 로그인(SSO) 세션이 유효한 상태에서만 가능합니다(3장 참고).
- 유휴 자동 인덱싱이 실제 PC 사용 여부를 더 정확히 감지하도록 개선됐습니다.

## 1. 설치

1. 받은 `AegisDesk.zip`을 원하는 위치에 압축 해제 (예: `C:\Tools\AegisDesk\`)
2. `AegisDesk\AegisDesk.exe` 더블클릭으로 실행
3. 최초 실행 시 온보딩 화면에서 인덱싱할 폴더를 지정하고 [인덱싱 시작]

별도 설치·관리자 권한·파이썬 설치가 필요 없습니다.

## 2. 기본 사용법

- 사이드바에서 검색 범위(내 PC 문서 / 공유 폴더 / 내 메일함)를 선택하고 질문을 입력합니다.
- [폴더 관리]에서 인덱싱 대상 폴더를 추가·제거할 수 있습니다.
- [지금 재인덱싱]으로 수동 갱신, 또는 자리를 비우면 자동으로 유휴 인덱싱이 동작합니다.

## 3. 알려진 제약사항

| 항목 | 내용 |
|---|---|
| **M:\ 드라이브** | EFSS2 DRM 가상 드라이브라 접근 불가(구조적 제약). 대신 K:\ 등 일반 공유 드라이브는 정상 동작 |
| **나스카 DRM 문서** | SSO 로그인이 유지된 상태에서만 복호화되어 인덱싱됩니다. 장시간 미사용으로 로그아웃되면 해당 사이클에서 건너뛰고, 다음 로그인 후 사이클에서 자동 재시도됩니다 |
| **doc/xls/ppt(구 버전 오피스 포맷) 및 DRM 보호된 docx/xlsx/pptx** | PC에 Microsoft Office가 설치되어 있어야 정상 파싱됩니다(구형 포맷은 항상, 최신 포맷은 DRM으로 보호된 경우에만 Office를 거쳐 읽습니다). Office는 PC당 하나의 인스턴스만 실행되는 구조라 사용자가 같은 앱을 열어두면 충돌할 수 있습니다. **이를 막기 위해, 해당 앱이 실행 중이면 그 형식(예: Excel 실행 중 → `.xls`/DRM `.xlsx`)만 이번 인덱싱 사이클에서 자동으로 건너뛰고 다음 사이클에서 다시 시도합니다.** 따라서 Office를 열어두어도 앱이 멈추지는 않지만, **열어둔 형식의 문서는 그동안 검색되지 않습니다.** DRM 보호가 없는 일반 docx/xlsx/pptx는 Office 없이 직접 파싱하므로 이 제약과 무관합니다 |
| **DRM 문서와 자리 비움(유휴)** | DRM으로 보호된 문서는 DRM/SSO 로그인 세션이 유효할 때만 인덱싱됩니다. 자리를 8분 이상 비우면 세션이 만료됐을 것으로 보고 DRM 문서만 그 사이클에서 건너뛰고, 자리로 돌아와 활동을 재개하면 곧바로 캐치업 인덱싱이 실행됩니다. DRM이 아닌 일반 문서는 이 제약과 무관하게 계속 인덱싱됩니다 |
| **첫 인덱싱 시간** | 폴더 규모·네트워크 상태에 따라 오래 걸릴 수 있습니다. 진행 중에는 사이드바에 실시간 진행률이 표시됩니다 |
| **창 닫기(X) 동작** | 앱이 종료되지 않고 **시스템 트레이로 숨겨집니다** (알림 영역, 시계 옆 ▲ 아이콘). 완전 종료는 트레이 아이콘 우클릭 → [종료]를 사용하세요 |
| **메일 인덱싱** | Knox `.mysingle`과 표준 `.eml` 파일을 지원합니다. Outlook `.msg`/PST는 아직 지원하지 않습니다 |

## 4. 문제가 발생했을 때

**구형 doc/xls/ppt 문서가 검색되지 않는 경우**: 해당 Office 앱(Word·Excel·PowerPoint)을
열어둔 동안에는 그 형식의 구형 문서를 자동으로 건너뜁니다(위 3장 참고 — 충돌 방지). 앱을 닫은 뒤
다음 인덱싱 사이클(유휴 자동 또는 [지금 재인덱싱])이 돌면 자동으로 다시 인덱싱되어 검색됩니다.
최신 포맷(docx/xlsx/pptx)은 이 제약과 무관합니다.

1. 앱을 재현 가능한 상태로 만들어 주세요 (어떤 질문/조작을 했을 때 발생했는지)
2. 아래 경로의 로그 파일을 확인·첨부해 주세요:
   ```
   %APPDATA%\AegisDesk\logs\aegisdesk.log
   ```
   (탐색기 주소창에 `%APPDATA%\AegisDesk\logs`를 붙여넣으면 바로 이동합니다)
3. 로그에는 파일 경로·건수·소요시간만 남고 **문서·메일 본문 내용은 기록되지 않습니다** — 안심하고 공유해 주세요.
