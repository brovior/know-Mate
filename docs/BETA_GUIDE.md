# Aegis Desk 베타 테스터 가이드

> 버전: 0.9.0-beta1

## 1. 설치

1. 받은 `AegisDesk.zip`을 원하는 위치에 압축 해제 (예: `C:\Tools\AegisDesk\`)
2. `AegisDesk\AegisDesk.exe` 더블클릭으로 실행
3. 최초 실행 시 온보딩 화면에서 인덱싱할 폴더를 지정하고 [인덱싱 시작]

별도 설치·관리자 권한·파이썬 설치가 필요 없습니다.

## 2. 기본 사용법

- 사이드바에서 검색 범위(내 PC 문서 / 공유 폴더 / 내 메일함)를 선택하고 질문을 입력합니다.
- [폴더 관리]에서 인덱싱 대상 폴더를 추가·제거할 수 있습니다.
- [지금 재인덱싱]으로 수동 갱신, 또는 자리를 비우면 자동으로 유휴 인덱싱이 동작합니다.

## 3. 알려진 제약사항

| 항목 | 내용 |
|---|---|
| **M:\ 드라이브** | EFSS2 DRM 가상 드라이브라 접근 불가(구조적 제약). 대신 K:\ 등 일반 공유 드라이브는 정상 동작 |
| **나스카 DRM 문서** | SSO 로그인이 유지된 상태에서만 복호화되어 인덱싱됩니다. 장시간 미사용으로 로그아웃되면 해당 사이클에서 건너뛰고, 다음 로그인 후 사이클에서 자동 재시도됩니다 |
| **doc/xls/ppt(구 버전 오피스 포맷)** | PC에 Microsoft Office가 설치되어 있어야 정상 파싱됩니다. ⚠ **인덱싱 중에는 Word·Excel·PowerPoint를 직접 열어두지 마세요.** 백그라운드 인덱싱이 같은 앱을 자동 제어하는데, Office는 PC당 하나의 인스턴스만 실행되는 구조라 사용자가 열어둔 창에 자동화가 그대로 붙어 충돌합니다. 이 상태에서 저장 확인 등 대기가 걸리면 Office가 응답 없음 상태로 멈출 수 있습니다. 인덱싱이 끝난 뒤 열거나, 인덱싱 전에 닫아주세요 |
| **첫 인덱싱 시간** | 폴더 규모·네트워크 상태에 따라 오래 걸릴 수 있습니다. 진행 중에는 사이드바에 실시간 진행률이 표시됩니다 |
| **창 닫기(X) 동작** | 앱이 종료되지 않고 **시스템 트레이로 숨겨집니다** (알림 영역, 시계 옆 ▲ 아이콘). 완전 종료는 트레이 아이콘 우클릭 → [종료]를 사용하세요 |
| **메일 인덱싱** | Knox `.mysingle`과 표준 `.eml` 파일을 지원합니다. Outlook `.msg`/PST는 아직 지원하지 않습니다 |

## 4. 문제가 발생했을 때

**Word/Excel/PowerPoint가 갑자기 사라지거나 응답 없음 상태가 된 경우**: 인덱싱 중 해당 문서를 직접
열어두었을 가능성이 높습니다(위 3장 참고). 강제 종료 후 재시작하면 되고, 인덱스나 문서 파일은
손상되지 않습니다. 재발 방지를 위해 인덱싱 중에는 doc/xls/ppt 파일을 열지 말아주세요.

1. 앱을 재현 가능한 상태로 만들어 주세요 (어떤 질문/조작을 했을 때 발생했는지)
2. 아래 경로의 로그 파일을 확인·첨부해 주세요:
   ```
   %APPDATA%\AegisDesk\logs\aegisdesk.log
   ```
   (탐색기 주소창에 `%APPDATA%\AegisDesk\logs`를 붙여넣으면 바로 이동합니다)
3. 로그에는 파일 경로·건수·소요시간만 남고 **문서·메일 본문 내용은 기록되지 않습니다** — 안심하고 공유해 주세요.
